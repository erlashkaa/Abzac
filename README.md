# Abzatz — Информационная система электронной библиотеки

## Описание проекта
**Abzatz** — современное веб-приложение для цифровой библиотеки, которое предоставляет читателям удобную среду для поиска, приобретения и интерактивного чтения книг, а авторам и администраторам — мощные инструменты управления контентом и модерации сообщества.

Информационная система спроектирована с акцентом на безопасность финансовых транзакций, социальное взаимодействие читателей и бесшовный пользовательский опыт при чтении книг непосредственно в веб-интерфейсе.

---

## Возможности
* **Безопасная аутентификация и авторизация**:
  * Использование механизма двойных JWT-токенов (`access` и `refresh`).
  * Бесшовное автоматическое продление сессий на клиенте (**Silent Refresh**) без прерывания пользовательского взаимодействия.
  * Криптографическое хеширование паролей с помощью алгоритма `BCrypt` с добавлением соли на стороне сервера.
  * Ролевая модель доступа со строгим разграничением прав на защищенные маршруты: **Гость**, **Читатель** (авторизованный пользователь), **Администратор**.
* **Витрина книг и умный поиск**:
  * Публичный каталог книг, доступный неавторизованным пользователям (гостям).
  * Полнотекстовый поиск произведений по названию и авторам, гибкая фильтрация и сортировка.
  * Просмотр карточек книг с подробным описанием, рейтингом, отзывами и возможностью чтения бесплатного ознакомительного фрагмента.
* **Транзакционная система покупок**:
  * Личный счет (виртуальный баланс) читателя с возможностью пополнения.
  * Списание средств при покупке книг в рамках единой ACID-транзакции на уровне базы данных (`@Transactional`), что гарантирует защиту от состояния гонки (Race Condition) и исключает потерю средств.
* **Интерактивный веб-ридер**:
  * Удобный интерфейс чтения с индивидуальной настройкой шрифта (размер, семейство) и цветовой темы оформления (светлая, темная, сепия).
  * Автоматический фоновый трекинг прогресса: сохранение текущей посещенной страницы и точного процента прочитанного в реальном времени при скролле.
  * Персональная система закладок для сохранения важных цитат и страниц в один клик.
* **Социальное взаимодействие**:
  * Публикация подробных рецензий к прочитанным произведениям с выставлением оценок по пятибалльной шкале.
  * Форум с древовидными иерархическими обсуждениями на основе рекурсивных связей «один-ко-многим» в БД (ответы на конкретные комментарии).
  * Система жалоб (Reports) для отправки пользователями репортов на спам и некорректное поведение в панель модерации.
* **Панель администратора**:
  * Аналитический дашборд со сводной статистикой (активные пользователи, продажи, популярные книги).
  * Управление цифровым каталогом (добавление, редактирование информации, удаление книг).
  * Модераторская панель для оперативной обработки жалоб (Reports) на рецензии и сообщения форума с возможностью их удаления.
  * Инструменты администрирования пользователей и блокировка учетных записей нарушителей правил сообщества.

## Структура проекта
Проект разделен на две изолированные части: клиентское SPA-приложение на React и серверную часть на Spring Boot.

### Backend (Серверная часть)
Размещен в директории `backend-java/` и построен в соответствии с многоуровневой архитектурой (Tiered Architecture).

```
backend-java/
├── pom.xml                                   # Конфигурация зависимостей Maven
└── src/main/java/ru/biblioteka/api/          # Исходный код бэкенда
    ├── config/                               # Конфигурационные файлы Spring Boot, настройки CORS
    ├── controller/                           # REST-контроллеры (HTTP эндпоинты API)
    ├── dto/                                  # Объекты переноса данных (Data Transfer Objects)
    ├── entity/                               # Сущности базы данных (JPA-модели ORM Hibernate)
    ├── mapper/                               # Мапперы (преобразование Entity <-> DTO)
    ├── repository/                           # Репозитории (Spring Data JPA интерфейсы к PostgreSQL)
    ├── security/                             # Конфигурация Spring Security, JWT-фильтры
    ├── service/                              # Бизнес-логика приложения (сервисные классы)
    ├── util/                                 # Вспомогательные классы и утилиты
    └── BibliotekaApiApplication.java         # Точка запуска Spring Boot приложения
```

### Frontend (Клиентская часть)
Размещен в корневой директории проекта и представляет собой SPA на базе React 18, Vite и TypeScript.

```
src/
└── app/
    ├── api/                                  # Клиентские HTTP-запросы (настройки Axios, интерцепторы JWT)
    ├── components/                           # Переиспользуемые UI-компоненты (Header, Footer, BookCard, Rating)
    │   ├── ui/                               # Низкоуровневые примитивы интерфейса
    │   └── figma/                            # Компоненты макета дизайна
    ├── context/                              # Глобальный React Context (авторизация, состояние сессии)
    ├── data/                                 # Статические демонстрационные данные
    ├── pages/                                # Страницы приложения
    │   ├── Admin.tsx                         # Панель управления администратора
    │   ├── Auth.tsx                          # Страницы входа и регистрации пользователей
    │   ├── BookDetails.tsx                   # Детальная страница книги с рецензиями
    │   ├── Catalog.tsx                       # Публичный каталог книг с поиском и фильтрами
    │   ├── Forum.tsx                         # Главная страница форума обсуждений
    │   ├── ForumTopic.tsx                    # Страница конкретной ветки форума с древовидными комментариями
    │   ├── Home.tsx                          # Главная приветственная страница
    │   ├── Profile.tsx                       # Личный кабинет читателя (баланс, купленные книги)
    │   ├── Reader.tsx                        # Интерактивный ридер с настройками и закладками
    │   ├── UserProfile.tsx                   # Просмотр публичного профиля другого читателя
    │   └── Privacy.tsx / Terms.tsx           # Страницы юридических соглашений и правил
    ├── App.tsx                               # Маршрутизация приложения (React Router, Protected Routes)
    └── main.tsx                              # Точка монтирования React-приложения к DOM
styles/                                       # Стилизация приложения (конфигурация Tailwind CSS)
index.html                                    # Корневой HTML-шаблон
vite.config.ts                                # Конфигурация сборщика Vite
package.json                                  # Управление зависимостями и скриптами сборки React-клиента
```

## Введение в эксплуатацию

### Требования к окружению
Для локального запуска и разработки проекта необходимы следующие установленные инструменты:
* **Java Development Kit (JDK)** версии 17
* **Node.js** версии 18 или выше (совместно с пакетным менеджером `npm`)
* **Apache Maven** (для сборки backend)
* **СУБД PostgreSQL** (версии 14 и выше)

---

## Настройка окружения

1. **Подготовка базы данных**:
   * Запустите локальный сервер PostgreSQL.
   * Создайте пустую базу данных с именем `abzatz`. Все таблицы и связи будут сгенерированы автоматически при первом старте бэкенда благодаря ORM Hibernate.
     ```sql
     CREATE DATABASE abzatz;
     ```

2. **Переменные окружения**:
   * Создайте файл `.env` в корневой директории проекта на основе примера `.env.example`.
   * Заполните параметры подключения к базе данных и настройки JWT:
     ```env
     # Подключение к PostgreSQL
     DB_HOST=localhost
     DB_PORT=5432
     DB_NAME=abzatz
     DB_USER=postgres
     DB_PASSWORD=your_secure_password

     # Настройки безопасности (JWT)
     JWT_SECRET=your_super_secret_signing_key_must_be_long_enough_to_be_secure
     JWT_ACCESS_EXPIRATION=900000          # 15 минут в миллисекундах
     JWT_REFRESH_EXPIRATION=604800000      # 7 дней в миллисекундах

     # Порты запуска компонентов
     BACKEND_PORT=8080
     FRONTEND_PORT=3000
     ```

## Запуск приложения

Программный комплекс развертывается и запускается локально без использования контейнеризации. Запуск выполняется в два этапа.

### Шаг 1: Запуск серверной части (Backend)
1. Откройте терминал и перейдите в каталог `backend-java`:
   ```bash
   cd backend-java
   ```
2. Выполните сборку проекта и запуск встроенного сервера Tomcat с помощью плагина Maven:
   ```bash
   mvn spring-boot:run
   ```
3. После успешного старта бэкенд будет доступен для обработки запросов по адресу:
   ```
   http://localhost:8080
   ```
   *При первом запуске Hibernate автоматически выполнит DDL-миграции, создав структуру таблиц в базе данных `abzatz`.*

### Шаг 2: Запуск клиентской части (Frontend)
1. Откройте второй терминал в корневой директории проекта.
2. Установите необходимые пакетные зависимости, описанные в `package.json`:
   ```bash
   npm install
   ```
3. Запустите сервер разработки Vite с поддержкой горячей замены модулей (HMR):
   ```bash
   npm run dev
   ```
4. После инициализации клиентская часть будет доступна в браузере по адресу:
   ```
   http://localhost:3000
   ```
   *(Или по адресу `http://localhost:5173` в зависимости от конфигурации портов в системе).*

## Руководство пользователя

### Авторизация и регистрация
* **Гостевой доступ**: При открытии приложения неавторизованному пользователю доступен основной функционал витрины — просмотр каталога книг, полнотекстовый поиск, переход на карточку произведения и чтение бесплатного ознакомительного фрагмента.
* **Регистрация**: Для получения доступа к личному кабинету и чтению книг полностью необходимо создать аккаунт. Перейдите на страницу регистрации, заполните имя пользователя, email и пароль. Пароль шифруется на сервере алгоритмом `BCrypt`.
* **Авторизация**: Выполните вход, введя email и пароль. Система сгенерирует пару JWT. Доступ к защищенным страницам (личный кабинет, чтение книг, панель администратора) строго разграничивается React Router на основе вашей роли. При истечении времени жизни `access`-токена интерцептор Axios автоматически обновит сессию в фоне (**Silent Refresh**), используя `refresh`-токен.

### Главный экран и каталог книг
* **Витрина (Каталог)**: Отображает список всех доступных в библиотеке произведений с удобной сеткой карточек.
* **Поиск и фильтрация**: Сверху расположена панель поиска. Введите ключевые слова (название книги или автора) для мгновенной фильтрации контента. Доступна сортировка книг по рейтингу, новизне или цене.
* **Карточка книги**: Клик по книге открывает страницу детального просмотра с аннотацией, рейтингом, списком отзывов и кнопками взаимодействия.

### Покупка книг и баланс
* **Личный кабинет**: На странице профиля читатель может увидеть свой текущий виртуальный баланс. Доступна форма пополнения счета.
* **Процесс покупки**: Если книга платная и еще не приобретена, на странице книги отображается кнопка «Купить» с указанием цены. При покупке баланс проверяется сервером, списывается стоимость книги, а сама книга навсегда добавляется в личную библиотеку пользователя. Операция выполняется транзакционно (ACID) с блокировкой от Race Condition.

### Интерактивный веб-ридер
* **Вход в ридер**: Купленная книга становится доступна для чтения. Клик по кнопке «Читать» запускает интерактивный веб-ридер.
* **Настройка интерфейса**: В верхней панели ридера доступны настройки стиля текста. Вы можете настроить:
  * Размер шрифта (увеличить/уменьшить);
  * Шрифт (Serif, Sans-Serif, Monospace);
  * Цветовую схему (Дневная / Ночная темная / Сепия для снижения нагрузки на глаза).
* **Автосохранение прогресса**: Во время чтения при прокрутке страниц клиент автоматически фиксирует текущую страницу и отправляет данные на сервер. Вы можете закрыть вкладку в любой момент — при следующем открытии ридера чтение начнется ровно с того места, где вы остановились.
* **Закладки**: Нажмите на иконку закладки в правом верхнем углу, чтобы сохранить текущую страницу. Список ваших закладок доступен в боковой панели ридера для быстрого перехода.

### Социальный блок (Отзывы и Форум)
* **Рецензии**: На странице книги авторизованные пользователи могут написать подробный отзыв о произведении и выставить оценку от 1 до 5 звезд. Рейтинг книги пересчитывается автоматически.
* **Форум обсуждений**: Раздел «Форум» содержит темы для обсуждения литературы и работы библиотеки. Внутри темы пользователи могут общаться в формате древовидных комментариев — отвечать на конкретные сообщения, выстраивая иерархическую структуру ответов.
* **Жалобы (Reports)**: Если вы заметили спам, рекламу или оскорбительный комментарий на форуме либо в рецензиях, нажмите кнопку «Пожаловаться». Жалоба мгновенно отправится в панель администратора.

### Панель администратора
Доступна только пользователям со специальной ролью `ADMIN`.
* **Дашборд**: Отображает текущую статистику по зарегистрированным пользователям, купленным книгам и общему балансу системы.
* **Управление каталогом**: Форма добавления новых книг (загрузка метаданных, текстового содержимого, обложки, автора, цены и бесплатного фрагмента), а также редактирование существующих произведений.
* **Модерация контента**: Раздел обработки жалоб (Reports). Администратор может просмотреть текст нарушения, перейти к родительской сущности и при необходимости удалить оскорбительный комментарий/рецензию.
* **Блокировка нарушителей**: Администратор имеет возможность временно или навсегда заблокировать учетную запись любого пользователя. Заблокированный пользователь теряет возможность авторизации в системе.
